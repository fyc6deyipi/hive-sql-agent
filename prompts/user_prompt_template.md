# LLM 节点（节点 4）用户提示词

> 精简版：CHK 走查规则已在 system prompt 中完整定义，用户提示词仅提供输入数据 + 输出指令。
> KB 检索已证明对强模型无正面贡献（A/B 测试净贡献 -0.10），已移除。
> 千帆控制台变量引用语法：`{{节点N.字段}}`。变量名以你控制台实际显示为准。
> 这份是要**复制粘贴到 LLM 节点用户提示词输入框**的最终版本。

```
<input>
sql:
{{节点1.sql}}

tables:
{{节点2.tables_md}}
</input>

按系统提示词中的 26 项 Checklist 逐项走查，命中的生成 issue，不命中的跳过。仅输出 JSON，不要 ``` 包裹，不要 Markdown 标题前缀。
```

## 变量引用注意事项

1. 两个变量 `{{节点1.sql}}` / `{{节点2.tables_md}}` 必须用千帆**变量选择器**点选（输入 `{{` 弹自动提示），不要手输。
2. 若仍保留节点 3 KB 检索：可在 `</input>` 后加 `<knowledge>{{节点3.OutputList}}</knowledge>`，但 A/B 测试表明对强模型无正面效果。
